# 📐 日报编辑质量规范

> 来源：对比 CodeBuddy 与 Claude Code 使用相同工具链产出的日报差异后，总结的四条核心改进。
> 这些规则是**硬性红线**，每次生成日报时必须逐条检查。

---

## 规则 1：搜索阶段强制英文优先

**问题**：中文搜索容易命中新浪/搜狐等二手转载源，导致信息失真、来源重复。

**规则**：
- 每条新闻**至少搜一次英文关键词**，确保命中一手源
- Tavily 搜索时，优先使用英文 query（如 `"Anthropic Claude lawsuit"` 而非 `"Anthropic 诉讼"`）
- 中文搜索仅作补充，用于获取国内独家新闻（如政策、国产芯片、国内公司动态）

**执行方式**：
```
# 正确 ✅
tavily_search("OpenAI GPT-5 release", search_depth="advanced")
tavily_search("OpenAI GPT-5 发布 国内反应", search_depth="basic")  # 补充

# 错误 ❌
tavily_search("OpenAI GPT-5 发布", search_depth="advanced")  # 仅中文
```

---

## 规则 2：信源红线

**问题**：过度依赖少数二手源（如新浪科技、腾讯科技），导致日报看起来像"中文科技门户摘要汇编"。

**规则**：
- 同一**二手源**（新浪科技、搜狐科技、腾讯科技、网易科技等）在整期日报中**引用不超过 2 次**
- **聚合博客**（如 Mean CEO Blog、AI新闻聚合号）**只做辅助参考**，不作为主引用源
- 每条新闻的 `来源` 字段必须标注**最上游的一手源**

**信源分级**：
| 级别 | 定义 | 使用规则 |
|------|------|----------|
| 🟢 一手源 | 官方博客、原始论文、发布会、当事公司 PR | 优先使用，不限次数 |
| 🟡 权威媒体 | TechCrunch、The Verge、Bloomberg、Reuters、CNBC | 优先使用，不限次数 |
| 🟠 二手媒体 | 新浪科技、搜狐、腾讯科技、36氪等转载型媒体 | 整期 ≤ 2 次引用 |
| 🔴 聚合/自媒体 | Mean CEO Blog、未署名公众号、营销号 | 仅辅助，禁止作为主源 |

---

## 规则 3：深度抓取不可省略

**问题**：仅凭搜索摘要（snippet）编写深度洞察，导致洞察空泛、缺乏事实支撑。

**规则**：
- **深度洞察**（insight-box）**必须基于 Jina Reader 或 tavily_extract 抓取的全文**撰写
- 禁止仅凭 Tavily/web_search 返回的 2-3 行摘要编写洞察
- 每期日报**至少深度抓取 3-5 篇高价值文章全文**

**执行方式**：
```
# 采集第 3 级（必做）
# 1. 从第 1 级筛出的 18-22 条中，选出 3-5 条最值得深挖的
# 2. 用 Jina Reader 抓全文
curl -s "https://r.jina.ai/https://techcrunch.com/2026/03/11/..."

# 3. 基于全文撰写洞察，洞察中应包含全文中的具体数据/引用
```

**洞察质量自检**：
- ✅ 好的洞察：引用了全文中的具体数据、CEO 原话、时间线对比
- ❌ 差的洞察：泛泛而谈"这将对行业产生深远影响"、"值得关注"

---

## 规则 4：信源多样性检查

**问题**：日报 footer 信源列表中英文一手源偏少，整体像中文翻译稿合集。

**规则（硬性红线）**：
| 指标 | 最低要求 |
|------|----------|
| 英文一手源 | **≥ 10** |
| 官方源（公司博客/PR/官方公告） | **≥ 3** |
| 全中文转载 | **禁止**（每条新闻必须能追溯到英文原始出处） |

**发布前检查清单**：
```
□ footer 信源列表中英文域名 ≥ 10 个（techcrunch.com, theverge.com, bloomberg.com 等）
□ 其中官方博客/PR ≥ 3 个（blog.openai.com, nvidia.com/blog, anthropic.com 等）
□ 没有"来源: 新浪科技"作为唯一来源的条目
□ 每条新闻都能追溯到英文一手源
```

---

## 总结：四条规则的执行时机

| 阶段 | 检查项 |
|------|--------|
| 搜索时 | 规则 1 — 每条新闻至少一次英文搜索 |
| 筛选时 | 规则 2 — 标记信源级别，控制二手源引用次数 |
| 撰写洞察时 | 规则 3 — 确认已抓取全文再写洞察 |
| 生成完成后 | 规则 4 — 核对 footer 信源多样性指标 |

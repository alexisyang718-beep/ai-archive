# 日报撰写检查清单

> 每次撰写日报前必须逐项确认，不可跳过！

## 阶段1：撰写前检查（必须）

### 1.1 执行信源检查脚本
```bash
python3 scripts/daily_brief_check.py YYYYMMDD
```
- [ ] 脚本执行通过，无错误
- [ ] 所有入选新闻都有有效来源
- [ ] 所有URL都可访问或已在采集数据中

### 1.2 确认新闻数量
- [ ] 总条数 25-30 条
- [ ] 每模块 4-6 条
- [ ] 头条 3 条已标记

### 1.3 确认一手官方源
- [ ] 官方一手源 ≥ 5 条（X官方账号、Bloomberg、Reuters等）

---

## 阶段2：撰写中检查（逐条）

### 每条新闻必须检查：

| 检查项 | 要求 | 工具 |
|--------|------|------|
| 正文长度 | 50-100 字 | 字数统计 |
| 洞察长度 | ≤ 300 字符 | 字符统计 |
| 头条洞察 | 250-300 字符 | 仅头条 |
| 数据标签 | 1-3 个 | 如"营收10亿"、"涨价15%" |
| 链接来源 | 与选题报告一致 | 核对JSON |

### 2.1 正文撰写规范
- [ ] 开头直接陈述核心事实
- [ ] 关键数据用 **加粗**
- [ ] 不添加个人观点或推测
- [ ] 不使用"据悉"、"据报道"等模糊表述

### 2.2 洞察撰写规范
- [ ] 必须基于全文，不是摘要
- [ ] 提供独特视角或深度分析
- [ ] 关联行业趋势或竞争格局
- [ ] 控制在 300 字符以内

---

## 阶段3：撰写后检查（必须）

### 3.1 内容检查
- [ ] 整期字数 8000-15000 字
- [ ] 无错别字
- [ ] 无格式错误（标签闭合、嵌套正确）
- [ ] 所有链接可点击

### 3.2 信源验证
```bash
# 检查是否有重复链接
python3 scripts/check_duplicate_links.py brief/YYYY-MM-DD.html

# 验证所有链接可访问
python3 scripts/verify_links.py brief/YYYY-MM-DD.html
```

### 3.3 模块分布检查
- [ ] 🤖 AI模型与产品：4-6 条
- [ ] 📱 手机与消费电子：4-6 条
- [ ] 🔧 芯片与算力：4-6 条
- [ ] 🎮 游戏行业：4-6 条
- [ ] 🏛️ 科技行业动态：4-6 条
- [ ] 📜 政策与监管：3-5 条

### 3.4 预览检查
- [ ] 本地启动 HTTP 服务器预览
- [ ] 手机端预览检查
- [ ] 邮件预览发送测试

---

- [ ] 已 commit

- [ ] 测试邮件已发送并检查

## 禁止事项（红线）

❌ **禁止仅凭搜索摘要写洞察**  
❌ **禁止使用无原始来源的新闻**  
❌ **禁止编造数据或事实**  
❌ **禁止超过 15000 字**  
❌ **禁止跳过检查清单**

## 快速命令

```bash
# 1. 信源检查
python3 scripts/daily_brief_check.py 20260328

# 2. 生成日报（示例）
python3 scripts/generate_daily.py 2026-03-28

# 3. 检查重复链接
python3 scripts/check_duplicate_links.py brief/2026-03-28.html

# 4. 发送测试邮件
python3 scripts/send_email.py brief/2026-03-28.html --to alexisyang@tencent.com

# 5. 发布到公众号
cd ../raphael-publish && node publish-daily.mjs ../tech-daily-brief/brief/2026-03-28.html nyt
```

# 科技资讯日报工作流

> 本目录存放新版工作流所需的配置文件和文档
> 采用**渐进式披露**设计，每个AI按步骤执行，知道该做什么、不该做什么

---

## 工作流概览

```
┌─────────────────────────────────────────────────────────────────┐
│  工作流1: 定时采集 (自动化)                                       │
│  输入: 定时触发                                                  │
│  输出: x.jsonl, weibo.jsonl, rss.jsonl                          │
│  AI角色: 监控状态、报告异常                                       │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  工作流2: 选题报告 (AI生成 → 人工确认)                            │
│  输入: 采集数据 + news-preferences.md                            │
│  输出: daily_selection_YYYYMMDD.md                              │
│  AI角色: 按偏好筛选、生成报告、等待确认                           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  工作流3: 撰写日报 (AI生成)                                       │
│  输入: 人工确认的选题 + template.html + writing-preferences.md   │
│  输出: brief/YYYY-MM-DD.html                                    │
│  AI角色: 抓取全文、撰写内容、格式化输出                           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  工作流4: 发布流程 (AI执行)                                       │
│  输入: 日报HTML文件                                              │
│  输出: GitHub + 邮件 + 公众号                                    │
│  AI角色: 按步骤执行发布                                          │
└─────────────────────────────────────────────────────────────────┘
```

## 渐进式披露原则

每个工作流的README和prompt.md都遵循：

1. **先读什么** - 明确列出必须读取的文件
2. **该做什么** - 清晰的任务描述
3. **怎么做** - 具体的执行步骤和标准
4. **不该做什么** - 明确的禁止事项
5. **输出什么** - 明确的输出要求

## 新闻偏好拆分说明

用户的新闻偏好已拆分为两个文件：

| 文件 | 工作流 | 内容 |
|------|--------|------|
| `news-preferences.md` | 2.选题 | 模块优先级、内容偏好、信源优先级、时效性、品牌平衡 |
| `writing-preferences.md` | 3.撰写 | 字数要求、洞察深度、信源权威性、原链接规则、格式规范 |

这样拆分的原因：
- 选题阶段关注"选什么"（内容、信源、时效）
- 撰写阶段关注"怎么写"（字数、深度、格式）

# AI 执行入口（防跑偏指南）

> ⚠️ **警告：不读此文件直接执行 = 100%跑偏**
> 
> 本文件是AI的"安全带"，强制按步骤执行，跳过任何一步都会触发检查点报错。

---

## 执行前强制检查清单

**开始前必须确认：**
- [ ] 我是否刚被唤醒，没有上下文记忆？
- [ ] 我是否只拿到了`workflow`文件夹？
- [ ] 我是否知道当前要执行哪个工作流？

**如果以上任一答案为"是" → 必须按本文件执行**

---

## 第一步：读取总览（强制）

**必须读取**：`workflow/README.md`

**为什么**：了解4个工作流的关系，确定当前该做哪个。

**检查点1**：读完后必须能回答：
- 当前是工作流几？
- 上一步应该输出什么文件？
- 下一步该读哪个文件夹？

**答不上来 → 重新读README**

---

## 第二步：进入对应工作流（强制）

根据用户指令确定工作流：

| 用户指令 | 对应工作流 | 进入文件夹 |
|---------|-----------|-----------|
| "看采集状态" | 工作流1 | `01-collection/` |
| "生成选题" / "看今天的选题" | 工作流2 | `02-selection/` |
| "写日报" / "生成日报" | 工作流3 | `03-generation/` |
| "发布" / "发邮件" / "发公众号" | 工作流4 | `04-publishing/` |

**检查点2**：必须读取对应文件夹的`README.md`

---

## 第三步：读取工作流README（强制）

**必须读取**：`workflow/XX-XXX/README.md`

**检查点3**：读完后必须能列出：
1. 该工作流要做什么？
2. 必须读取哪些文件？（按顺序）
3. 有没有脚本可以用？
4. 输出什么文件？

**答不上来 → 重新读工作流README**

---

## 第四步：检查脚本（强制）

**必须读取**：`workflow/scripts/README.md`

**检查点4**：确认当前工作流是否有对应的脚本：

| 工作流 | 对应脚本 | 是否必须先执行？ |
|-------|---------|---------------|
| 工作流2 | `01_preprocess_candidates.py` | ✅ 是 |
| 工作流3 | `02_fetch_fulltext.py` | ⚠️ 选题确认后执行 |
| 工作流4 | `03_publish_pipeline.py` | ✅ 是 |

**有脚本必须先执行脚本，不要重复做脚本的事**

---

## 第五步：读取偏好配置（强制）

**工作流2必须读**：`workflow/02-selection/news-preferences.md`

**工作流3必须读**：`workflow/03-generation/writing-preferences.md`

**检查点5**：读完后必须能回答：
- 用户偏好什么内容？
- 讨厌什么内容？
- 有什么硬性规则？（时效性/数量/品牌平衡）

**答不上来 → 重新读偏好配置**

---

## 第六步：读取Prompt（强制）

**必须读取**：对应工作流的`prompt.md`

**检查点6**：读完后必须知道：
- 具体执行步骤
- 禁止事项
- 输出格式

---

## 第七步：执行并检查输出（强制）

**每完成一步，检查是否符合预期**：

### 工作流2检查点
- [ ] 是否按6大模块分类？
- [ ] 每模块是否4-6条？
- [ ] 前5条是否标记✅？
- [ ] 是否包含URL、来源、入选理由？
- [ ] 手机模块是否至少3个品牌？
- [ ] 是否全部在48h内？

### 工作流3检查点
- [ ] 是否基于template.html？
- [ ] 每条是否有原始链接（去JSON提取的）？
- [ ] 洞察是否≤300字符？
- [ ] 正文是否50-100字？
- [ ] 是否有数据标签？

### 工作流4检查点
- [ ] GitHub是否push成功？
- [ ] 测试邮件是否发送？
- [ ] 是否等待人工确认？
- [ ] 群发是否执行？
- [ ] 公众号是否同步？

## 紧急制动：什么时候必须停止

遇到以下情况，**立即停止并告知用户**：

1. **工作流2**：预处理脚本输出为空（没有候选新闻）
2. **工作流2**：X采集数据为0（可能Cookie过期）
3. **工作流3**：选题报告未确认（用户没说过"确认"）
4. **工作流3**：template.html读不到
5. **工作流4**：日报文件不存在
6. **任何步骤**：不知道下一步该做什么

**停止话术**："我遇到了XX问题，需要确认：YY。请检查ZZ。"

## 记忆锚点

如果你发现自己：
- 不确定该做工作流几 → **回到第一步读README**
- 不确定用户偏好 → **回到第五步读preferences**
- 在做重复劳动 → **回到第四步检查脚本**
- 输出格式不对 → **回到第六步读prompt**

**永远不要凭感觉执行，永远按检查点一步步来。**

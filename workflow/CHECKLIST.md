## 工作流2：选题报告 专用检查

### 读取阶段
- [ ] 已读取 `news-preferences.md`
- [ ] 已读取 `prompt.md`
- [ ] 已检查 `candidates_YYYY-MM-DD.csv` 是否存在
- [ ] 如不存在，已执行 `01_preprocess_candidates.py`

### 生成阶段
- [ ] 是否按6大模块分类？
- [ ] 每模块是否4-6条？
- [ ] 前5条是否标记 ✅？
- [ ] 每条是否包含：标题、来源、URL、时间、核心内容、入选理由？
- [ ] 手机模块是否覆盖至少3个品牌？
- [ ] 是否全部在48h内（主体24h内）？
- [ ] 一手官方源是否≥5条？
- [ ] 同一二手源是否≤2条？

### 输出阶段
- [ ] 输出到 `v2/docs/daily_selection_YYYYMMDD.md`？
- [ ] 是否包含"今日三大头条"？
- [ ] 是否包含"等待人工确认"提示？

### 关键禁止项
- [ ] **禁止**不读preferences凭感觉选题
- [ ] **禁止**跳过预处理脚本直接读JSON
- [ ] **禁止**选超过48小时的旧闻
- [ ] **禁止**手机模块只选单一品牌
- [ ] **禁止**不写入选理由
- [ ] **禁止**不等待人工确认就进入工作流3

## 工作流3：撰写日报 专用检查

### 读取阶段
- [ ] 已读取 `writing-preferences.md`
- [ ] 已读取 `template.html`
- [ ] 已确认选题报告已人工确认（用户明确说过）
- [ ] 已检查 `fulltext_YYYY-MM-DD.json` 是否存在
- [ ] 如需要，已执行 `02_fetch_fulltext.py`

### 撰写阶段
- [ ] 是否基于template.html填充（非自行创建）？
- [ ] 是否只写人工确认的新闻（不自行添加）？
- [ ] 每条标题是否≤30字？
- [ ] 每条正文是否50-100字？
- [ ] 每条洞察是否≤300字符？
- [ ] 洞察是否基于全文+权威观点（非仅凭摘要）？
- [ ] 每条是否有1-3个数据标签？
- [ ] 每条是否有可点击的原始链接（去JSON提取的）？
- [ ] 是否标注信源类型（一手官方/一手媒体/二手编译）？

### 格式检查
- [ ] 6大板块顺序是否正确？
- [ ] 一手官方源是否≥5条？
- [ ] 同一二手源是否≤2次？
- [ ] 文件大小是否在30KB-100KB？

### 输出阶段
- [ ] 输出到 `brief/YYYY-MM-DD.html`？
- [ ] 是否能在浏览器正常打开？

### 关键禁止项
- [ ] **禁止**不读preferences凭感觉写
- [ ] **禁止**不读template.html自行写HTML
- [ ] **禁止**选题未确认就写日报
- [ ] **禁止**丢失原始链接（必须去JSON提取）
- [ ] **禁止**洞察仅凭搜索摘要写
- [ ] **禁止**空洞描述（无数据/事实支撑）

## 紧急停止信号

出现以下情况，**立即停止并报告用户**：

- [ ] 预处理脚本输出为空（无候选新闻）
- [ ] X采集数据为0（Cookie可能过期）
- [ ] 选题报告未确认（用户没说"确认"）
- [ ] template.html读不到
- [ ] 日报文件不存在
- [ ] 任何检查点无法通过

**停止话术模板**：
```
我遇到了[问题]，需要您的确认：
[具体描述]

可能原因：
1. [原因1]
2. [原因2]

建议操作：
- [操作1]
- [操作2]

请确认后我继续。
```
